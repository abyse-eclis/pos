# QuickPOS

ระบบ POS สำหรับหน้าร้านที่แยกงานออกเป็น 3 ส่วนหลัก:

- งานขายหน้าร้าน
- งานสต๊อกแบบรอบขาย (`session`)
- งานจัดการรูปสินค้าแบบไม่ใช้ฐานข้อมูลเพิ่ม

เวอร์ชันนี้ออกแบบให้ใช้งานผ่าน `responsive sidebar` และ deploy บน Vercel ได้ โดยใช้:

- Google Sheet เป็น data store หลัก
- Vercel Blob สำหรับเก็บรูปสินค้า

## ฟีเจอร์หลัก

- `ขายสินค้า`
  - เลือกสินค้าใส่ตะกร้า
  - ชำระเงินสดหรือโอน
  - บันทึกยอดขายรายวัน
- `สต๊อก`
  - เปิดรอบขาย
  - รับเข้าคลังร้าน
  - เบิกไปหน้าร้านหลายรอบ
  - เปรียบเทียบคลังร้าน/หน้าร้าน/ยอดขาย
  - ปิดรอบและคืนของเข้าคลัง
- `จัดการสินค้า`
  - อัปโหลดรูปสินค้าไป Vercel Blob
  - บันทึก URL รูปกลับลง Google Sheet
- `นับเงิน`
  - เงินทอนตั้งต้น
  - ยอดขายเงินสด
  - ยอดโอนแบบ manual-first
  - นับเงินจริงและส่วนต่าง
- `สรุปยอด`
  - ยอดรวมรายวัน
  - เงินสด / เงินโอน
  - รายการสินค้า
  - รายการบิล
- `คู่มือ`
  - อธิบายว่าแต่ละหน้าอยู่ตรงไหน
  - มีปุ่มลัดไปหน้าใช้งานจริง

## Flow การใช้งาน

### 1. ขายสินค้า

- เข้า `/`
- เลือกสินค้าใส่ตะกร้า
- ชำระเงินสดหรือโอน
- ระบบบันทึกยอดขายรายวัน

### 2. เปิดรอบขายสต๊อก

- เข้า `สต๊อก > คลังร้าน`
- สร้างรอบขายใหม่ เช่น `รอบตลาดศุกร์-เสาร์-อาทิตย์`
- หนึ่งรอบขายสามารถกินหลายวันได้

### 3. รับเข้าคลังร้าน

- ใช้หน้า `สต๊อก > คลังร้าน`
- กรอกจำนวนที่รับเข้าจริงจากคลังใหญ่
- ระบบนี้ track เฉพาะ `คลังร้าน` และ `หน้าร้าน`

### 4. เบิกไปหน้าร้าน

- ใช้หน้า `สต๊อก > หน้าร้าน`
- เบิกจากคลังร้านไปหน้าร้านได้หลายรอบต่อวัน
- ระบบบันทึกเป็น movement log

### 5. เปรียบเทียบสต๊อก

- ใช้หน้า `สต๊อก > เปรียบเทียบ`
- ดูต่อสินค้า:
  - ยกมาคลัง
  - รับเข้า
  - เบิกไปหน้าร้าน
  - คืนเข้าคลัง
  - คลังคงเหลือ
  - หน้าร้านคงเหลือ
  - ยอดขายวันนี้
  - หน้าร้านควรเหลือ
  - นับจริง
  - ส่วนต่าง

### 6. ปิดรอบขาย

- ใช้หน้า `สต๊อก > ปิดรอบ`
- นับหน้าร้านจริง
- คืนของกลับเข้าคลังร้าน
- ปิดรอบเมื่อเคลียร์ครบแล้ว

### 7. จัดการรูปสินค้า

- ใช้หน้า `สินค้า`
- เลือกสินค้า
- อัปโหลดรูปใหม่ขึ้น Vercel Blob
- ระบบบันทึก URL ลงคอลัมน์ `image` ใน Google Sheet อัตโนมัติ

### 8. นับเงินรายวัน

- ใช้หน้า `นับเงิน`
- กรอก:
  - เงินทอนตั้งต้น
  - ยอดโอนจริงจากสลิป
  - จำนวนธนบัตรและเหรียญ
- ระบบคำนวณยอดควรมีและส่วนต่าง

### 9. ดูสรุปยอดรายวัน

- ใช้หน้า `สรุปยอด`
- เลือกวันที่
- ดูยอดรวม รายการสินค้า และรายการบิล

## เส้นทางสำคัญ

- `/` หน้า POS หลัก
- `/stock/warehouse` คลังร้าน
- `/stock/storefront` หน้าร้าน
- `/stock/comparison` เปรียบเทียบ
- `/stock/closeout` ปิดรอบ
- `/products` จัดการสินค้าและรูปสินค้า
- `/cash-count` นับเงินรายวัน
- `/summary` สรุปยอดรายวัน
- `/docs` คู่มือการใช้งาน

## โครงสร้างข้อมูลหลัก

### Daily flow

- ยอดขาย
- เงินสด
- เงินโอน
- นับเงินจริง

ทั้งหมดผูกกับ `รายวัน`

### Stock flow

สต๊อกผูกกับ `รอบขาย`

- `session`
  - `session_id`
  - `label`
  - `started_at`
  - `closed_at`
  - `status`
- `movement`
  - `session_id`
  - `timestamp`
  - `sku_code`
  - `movement_type`
  - `qty_piece`

movement หลักมี 4 แบบ:

- `receive_to_warehouse`
- `move_to_storefront`
- `return_to_warehouse`
- `storefront_count`

## Google Sheet

ระบบใช้ Google Sheets เป็น backend storage หลัก

sheet สำคัญ:

- `สินค้า`
- `รอบขายสต๊อก`
- `ความเคลื่อนไหวสต๊อก`
- `นับเงินรายวัน`
- sheet ยอดขายรายวันแบบ `DD-MM-YYYY`

สำหรับรูปสินค้า ระบบจะเขียน URL ลงคอลัมน์ `image`

## Environment Variables

สร้างไฟล์ `.env.local`

```env
GOOGLE_SERVICE_ACCOUNT_EMAIL=your-service-account@your-project.iam.gserviceaccount.com
GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nxxxxxx\n-----END PRIVATE KEY-----\n"
GOOGLE_SHEET_ID=your-google-sheet-id-here
BLOB_READ_WRITE_TOKEN=vercel_blob_rw_xxxxx
```

## Development

ติดตั้ง dependencies:

```bash
npm install
```

รัน dev server:

```bash
npm run dev
```

เปิดเบราว์เซอร์ที่:

```text
http://localhost:3000
```

## Build / Check

```bash
npm run build
cmd /c npx tsc --noEmit
```

## หมายเหตุ

- ถ้า deploy บน Vercel ไม่ควรอัปโหลดไฟล์ลง `public/` ตอน runtime
- รูปสินค้าควรเก็บใน Vercel Blob แล้วบันทึก URL ลง Google Sheet
- หน้า `/products` ถูกออกแบบมาเพื่อเลี่ยงการต้องมีฐานข้อมูลแยกสำหรับ media
